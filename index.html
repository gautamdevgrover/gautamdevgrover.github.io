End-to-End CI/CD Pipeline
Jenkins · Docker · Helm · K8s

A production-grade CI/CD pipeline automating build, test, containerization, and deployment of a Node.js application — with multi-layer rollback protection and HTML email notifications.

GitHub → Jenkins → Docker Build → Test → Push → Helm Deploy → K8s
Jenkins Docker Kubernetes Helm Node.js GitHub

  • Post-deploy validation using kubectl rollout status

PROJECT_02

GitOps CI/CD Pipeline
Jenkins · Argo CD · Helm · K8s

A complete end-to-end GitOps pipeline where Jenkins builds and tests a Flask application, updates Helm chart values, and Argo CD ensures the Kubernetes cluster state always matches the desired configuration in Git.

Dev → GitHub → Jenkins → DockerHub → values.yaml → Argo CD → K8s
Jenkins Argo CD Helm Kubernetes Flask GitOps Kind
  • Full GitOps workflow — Git is the single source of truth
  • Jenkins builds & pushes Docker image, then updates Helm values.yaml in GitOps repo
  • Argo CD auto-detects Git changes and syncs to Kubernetes cluster
  • Automated testing via /health check before image push
  • Deployed on Kind cluster (EC2) with port-forward access
  • Separate app repo and manifest repo for clean separation of concerns
